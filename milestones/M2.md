Milestone 2 — Goals

--------------------

[] 1. Temporal Profile Extraction (Core Capability)

Goal: Every document has a usable, structured temporal profile.

Deliverables
Extract and store:
doc_type
document_date
event_date
coverage_start
coverage_end
effective_start
effective_end
due_date
temporal_confidence
temporal_source
Implement doc-type-specific rules:
statement → coverage_end / statement_as_of_date
invoice → service_date > invoice_date
minutes → meeting_date
contract → effective_start/end
service visit → event_date
Success Criteria
≥80% of docs get at least one correct primary date
No reliance on created_at for answers

--------------------

[] 2. Document Classification (Required for correctness)

Goal: Correctly identify document type before extraction.

Deliverables

Classifier (LLM or rules) → outputs:

{ "doc_type": "statement|invoice|minutes|contract|visit|other" }
Confidence score
Success Criteria
≥90% accuracy on test corpus
Misclassification is visible in UI (not silent)

--------------------

[] 3. Deterministic + NLP Date Extraction Pipeline

Goal: Robust multi-source date extraction.

Deliverables
Layered extractor:
Filename parser
Header/label parser
Metadata parser
NLP temporal parser (e.g. relative phrases)
Normalize to ISO dates
Store multiple candidates + confidence
Success Criteria
Handles:
“Jan 2026”
“Period Ending 1/31/26”
“Service Date: Feb 14, 2026”
Produces structured output, not raw text

--------------------

[] 4. Event Derivation Layer (Major Upgrade)

Goal: Convert documents → queryable events.

Deliverables

Event schema:

{
  "event_type",
  "event_date",
  "vendor",
  "source_doc_id",
  "confidence"
}
Extract events for:
service visits
invoices
inspections
meetings
Success Criteria
Query works without vector search:
“last irrigation visit”
≥70% of relevant docs produce usable events

--------------------


[] 5. Chunk Temporal Inheritance

Goal: Ensure chunks retain correct time context.

Deliverables
All chunks inherit:
doc_type
primary temporal fields
No chunk exists without temporal metadata
Success Criteria
Retrieval never returns a chunk with missing time context

--------------------

[] 6. Version / Supersession Tracking (Basic)

Goal: Avoid stale answers from outdated docs.

Deliverables
Fields:
is_latest_version
supersedes_document_id
Simple heuristic:
same doc_type + similar name → newer replaces older
Success Criteria
“most recent” never returns superseded docs

--------------------

[] 7. Ingest Pipeline (End-to-End)

Goal: Fully automated ingest pipeline.

Flow
upload
→ text extraction / OCR
→ doc classification
→ temporal extraction
→ event extraction
→ chunking + inheritance
→ store (docs + chunks + events)
Success Criteria
One command / API call processes a document completely
No manual steps required

  Endpoint:
  POST /ingest

  Initiate ingest of a drive tree.  Returns a poll URL.

  Endpoint:
  GET /ingest/progress/<uuid>?since=<uuid>

  Poll for ingest progress.

--------------------

[] 8. Demonstration UI (Critical)

Goal: Make ingest visible, inspectable, and debuggable.

Deliverables

A. Document Inspector View

For each document show:

Doc Type: Statement (confidence 0.94)

Temporal Profile:
- coverage_end: 2026-01-31 (0.95, header)
- document_date: 2026-02-03 (0.72, metadata)

Events:
- type: statement_generated
- date: 2026-01-31

Chunks:
- [chunk 1] inherits 2026-01-31

B. Extraction Trace

Show where each date came from
filename / header / NLP / metadata
C. Editable Overrides
User can fix:
doc_type
key dates
Overrides persist

--------------------

[] 9. Temporal Test Suite (Non-negotiable)

Goal: Prove correctness with real queries.

Test Queries
“most recent statement”
“statement from 5 years ago”
“last irrigation visit”
“next year budget”
Success Criteria
Correct document/event returned
Date reasoning is explainable

--------------------

[] 10. Metrics & Diagnostics

Goal: Make failures obvious.

Track:
% docs with valid primary date
classification accuracy
event extraction rate
temporal confidence distribution
Logging:
[INGEST]
doc=abc.pdf
type=statement (0.91)
coverage_end=2026-01-31 (header)
event=statement_generated
