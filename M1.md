Domifile — Milestone 1 (HOA Document Brain Demo)
================================================

## Objective

Build a working demo that connects to a Google Drive folder containing HOA documents and answers natural-language questions about those documents with cited sources.

The goal is to prove the core value proposition:

A board member can ask a question about the HOA and instantly find the answer from the document archive.

Accuracy does not need to be perfect. The success criterion is that the system reliably finds relevant documents and produces plausible answers with citations.

## Scope

Milestone 1 implements a simple RAG pipeline:

Google Drive
→ Document ingestion
→ Text extraction
→ Chunking + embeddings
→ Vector search
→ Answer generation
→ Chat interface

No knowledge graph or structured entity extraction is required yet.

## Core Components

Google Drive Connector

 * Connect Domifile to a Drive folder.

 * Functions:

   -- list files in a folder
   -- download files

Document Ingestion

 * Supported formats: PDF DOCX TXT

 * For each file:

   -- download file
   -- extract text
   -- store document metadata

 * Document fields:

   -- id
   -- drive_file_id
   -- filename
   -- text_content

Chunking

 * Split each document into chunks for retrieval.

 * Recommended parameters:

   -- chunk size: about 800 tokens
   -- overlap: about 100 tokens

 * Store each chunk in the database.

Embeddings

 * Generate an embedding vector for every chunk.

 * Store embeddings in Postgres with pgvector.

 * Chunk fields:

   -- id
   -- document_id
   -- text
   -- embedding

 * Create a vector similarity index.

Query Planner

 * Before searching, expand the user question into multiple search queries.

 * Example:

   User question:

     When was the irrigation system replaced?

   Planner output:

     irrigation replacement
     irrigation contract
     irrigation invoice
     irrigation maintenance
     irrigation installation

   Each query is then used to retrieve relevant document chunks.

Retrieval

  For each planned query:

    -- generate embedding
    -- perform vector search
    -- return top chunks

  Combine and deduplicate results.

Answer Generation

    Send the retrieved chunks to an LLM to generate the answer.

Prompt shape:

  "Answer the HOA question using the documents below.
  Cite the document names when possible."

  Inputs:

    -- question
    -- retrieved document chunks

  Outputs:

    -- answer
    -- sources

Chat API

  Endpoint:
  POST /ask

  Example request:

    question = "When was the pool resurfaced?"

  Example response:

    answer

    sources: ["pool_invoice_2019.pdf"]

Minimal UI

 * Single-page interface with:

    -- question input
    -- generated answer
    -- cited documents
    -- links to the Drive files

Technology Stack

LLM:

   -- a Drive folder can be connected
